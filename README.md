# Интеллектуальная система поддержки принятия решений для игры Dota 2

## 1. Общая характеристика проекта
Данный проект представляет собой интеллектуальную систему поддержки принятия решений (Intelligent Decision Support System), предназначенную для помощи игрокам в игре Dota 2 при выборе контр-пиков и предметных рекомендаций.

Проект разрабатывается в рамках учебного курса по интеллектуальным системам и направлен на формирование практических навыков полного цикла разработки ИС: от формализации знаний и проектирования архитектуры до реализации логики принятия решений и пользовательского интерфейса.

Система не вмешивается в игровой процесс и не взаимодействует с игровыми серверами. Все рекомендации формируются на основе заранее заданных экспертных правил и структурированной базы знаний.

---

## 2. Цель и задачи проекта

### Цель проекта
Разработка MVP интеллектуальной системы, способной анализировать состав вражеской команды и выдавать обоснованные рекомендации по контр-пикам и предметам на основе экспертных правил.

### Задачи проекта
- Формализовать знания о героях Dota 2 (роли, способности, уязвимости).
- Разработать rule-based ядро системы на основе правил if–then.
- Реализовать прототип пользовательского интерфейса (консольный интерфейс).
- Спроектировать архитектуру системы в виде UML-схемы.
- Подготовить проект к дальнейшему расширению (NLP, Computer Vision).

---

## 3. Область применения
Проект может использоваться в учебных целях как пример интеллектуальной системы поддержки принятия решений, а также как демонстрация гибридного подхода к разработке ИС (символьный AI + субсимвольный AI на следующих этапах).

---

## 4. Архитектура системы (MVP)

Система состоит из следующих логических компонентов:
- Пользователь
- Текстовый интерфейс (консоль)
- Rule-Based Engine (экспертная система)
- База знаний (JSON/CSV)
- Модуль формирования рекомендаций и объяснений

Подробная архитектурная схема приведена в папке `docs/`.

---

## 5. MVP (Модуль 1, Недели 1–5)

На текущем этапе реализуется MVP системы, основанный исключительно на жесткой логике и правилах (без использования машинного обучения).

### Входные данные:
- Список вражеских героев (ввод вручную через консоль)

### Выходные данные:
- Рекомендуемые контр-герои
- Рекомендуемые предметы
- Объяснение логики принятого решения

# Архитектура интеллектуальной системы

## 1. Общая архитектура
Архитектура системы построена по модульному принципу и соответствует требованиям курса по разработке интеллектуальных систем.

Система реализует полный цикл обработки данных:
Ввод → Анализ → Принятие решения → Объяснение результата.

---

## 2. Компоненты системы

### Пользователь
Пользователь взаимодействует с системой через текстовый интерфейс, вводя список вражеских героев.

### Текстовый интерфейс
Отвечает за:
- прием входных данных
- передачу данных в Rule Engine
- отображение рекомендаций пользователю

### Rule-Based Engine (Экспертная система)
Центральный компонент системы, реализующий логику принятия решений на основе правил if–then.

Пример правил:
- Если у врага много контроля, рекомендовать Black King Bar.
- Если у врага есть уклонение, рекомендовать Monkey King Bar.

### База знаний
Содержит структурированные данные:
- список героев
- теги героев (контроль, урон, уклонение и т.д.)
- контр-пики

База знаний хранится в формате JSON/CSV.

### Модуль рекомендаций
Формирует итоговый ответ системы:
- список контр-пиков
- список предметов
- объяснение причин рекомендаций

---

Архитектура (потоки данных)
flowchart TD
  U[Пользователь] -->|Ввод героев / текст / скрин| UI[UI: Console/Streamlit]

  UI -->|Список героев| RE[Rule Engine: if-then]
  UI -->|Текст| NLP[NLP модуль]
  UI -->|Скрин| PRE[OpenCV предобработка]
  PRE --> CV[CV модуль (классификация иконок)]
  CV -->|Распознанные герои| RE
  NLP -->|Сущности героев| RE

  RE --> KB[(База знаний JSON/CSV)]
  KB --> RE

  RE --> OUT[Рекомендации + объяснение]
  OUT --> UI
