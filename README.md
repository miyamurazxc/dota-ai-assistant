Интеллектуальная система поддержки принятия решений для игры Dota 2
1. Общая характеристика проекта

Данный проект представляет собой интеллектуальную систему поддержки принятия решений (Intelligent Decision Support System), предназначенную для помощи игрокам в игре Dota 2 при выборе контр-пиков и предметных рекомендаций.

Проект разработан в рамках дисциплины «Проектирование и разработка интеллектуальных систем» и направлен на формирование практических навыков полного цикла разработки интеллектуальной системы:

формализация знаний,

проектирование архитектуры,

реализация продукционной модели,

разработка пользовательского интерфейса,

интеграция элементов обработки естественного языка.

Система не взаимодействует с игровыми серверами и не вмешивается в игровой процесс. Все рекомендации формируются на основе экспертных правил и структурированной базы знаний.

2. Цель и задачи проекта
Цель проекта

Разработка MVP интеллектуальной системы, способной анализировать состав вражеской команды и выдавать обоснованные рекомендации по контр-пикам и предметам на основе экспертных правил.

Задачи проекта

Формализовать знания о героях Dota 2 (роли, теги, угрозы, уязвимости).

Разработать rule-based ядро системы на основе правил if–then.

Реализовать консольный пользовательский интерфейс.

Реализовать модуль объяснимых рекомендаций (Explainable AI).

Добавить модуль NLP для обработки текстовых запросов.

Спроектировать архитектуру системы в виде модульной UML-структуры.

Подготовить систему к дальнейшему расширению (CV, веб-интерфейс).

3. Область применения

Проект может использоваться:

как учебный пример интеллектуальной системы поддержки принятия решений,

как демонстрация символьного подхода (Rule-Based AI),

как основа для гибридной интеллектуальной системы (Symbolic + Sub-symbolic AI).

4. Архитектура системы
4.1 Общая архитектура (MVP)

Система построена по модульному принципу и реализует следующий цикл:

Ввод → Анализ → Принятие решения → Объяснение результата

Логические компоненты:

Пользователь

Текстовый интерфейс (CLI)

NLP-модуль

Rule-Based Engine (экспертная система)

База знаний (JSON/CSV)

Модуль рекомендаций и объяснений

4.2 Компоненты системы
1. Пользователь

Пользователь взаимодействует с системой через консольный интерфейс или текстовый ввод (NLP-режим).

2. Текстовый интерфейс (CLI)

Отвечает за:

прием входных данных (герои, роль),

передачу данных в ядро системы,

отображение рекомендаций.

Реализован в файле cli.py.

3. NLP-модуль (Week 6)

Позволяет системе обрабатывать свободный текст:

Пример запроса:

Я саппорт, против Storm Spirit и PA, у врагов много контроля и магии, что пикнуть?

Модуль:

определяет роль игрока,

извлекает имена героев (включая сокращения),

распознаёт угрозы (many_stuns, heavy_magic и т.д.),

преобразует текст в структурированный запрос.

Реализован в nlp.py.

4. Rule-Based Engine (Экспертная система)

Центральный компонент системы.

Использует продукционные правила вида:

Если у врага много контроля → рекомендовать BKB.

Если у врага Storm Spirit → рекомендовать instant disable.

Если не хватает фронтлайна → рекомендовать tank-героев.

Если против Phantom Assassin → рекомендовать Axe.

Ядро реализовано в rules.py.

Система начисляет баллы кандидатам и формирует объяснение причин выбора.

5. База знаний

Содержит:

heroes.json — информация о героях,

counters.csv — контр-пики,

items.json — соответствие «угроза → предмет».

База знаний является статической и может быть расширена.

6. Модуль рекомендаций

Объединяет:

правила,

данные,

контр-пики,

угрозы.

Формирует итоговый результат:

список героев,

список предметов,

объяснение логики принятия решения.

6. Расширенная архитектура (Перспектива развития)

В дальнейшем система может быть расширена следующим образом:

flowchart TD
U[Пользователь] -->|Текст| NLP[NLP-модуль]
U -->|Скриншот| PRE[OpenCV предобработка]

PRE --> CV[CV-модуль]
CV --> PARSE[Парсер героев]

NLP --> ANALYZE[Анализ угроз и состава]
PARSE --> ANALYZE

ANALYZE --> KB[База знаний]
ANALYZE --> RE[Rule-Based Engine]

RE --> OUT[Рекомендации]
OUT --> UI[Streamlit UI]

Планируется:

добавление Computer Vision для анализа скриншотов драфта,

внедрение веб-интерфейса (Streamlit),

возможное добавление ML-модели для ранжирования.

Система демонстрирует:

формализацию знаний,

построение продукционной модели,

реализацию объяснимых рекомендаций,

Проект является основой для дальнейшего развития гибридной интеллектуальной системы.
